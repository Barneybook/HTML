HTML

Taking a closer look at the document's body.

A simple, extensible way to traverse and use the DOM. HTML.min.js (1.2kb)

Input

Output

Transverse.the.DOM

HTML allows you to traverse the DOM via the dot operator like you would any Javascript object. This makes for a streamlined and native experience like it always should have been.

Element creation

One of the most useful components of HTML is its element creation interface. Using some Javascript getter magic, you can create and append elements by simply naming and chaining them.

Always an element

Whenever you're dealing with HTML, it's always a node or an array of nodes. One of the main goals of HTML was to abstract as little as possible from the DOM and keep things clean.

FAQ

Which browsers does HTML support?

HTML supports all major browsers.

How does it work?

HTML works by utilizing object getter functions to expose an element's children via their tag names on itself.

What?! You expose every element's children onto itself?

Nope! An element's children is only exposed when it's required so only the elements used will be HTML'fied.

Does that pollute the element's object with lots of properties?

You'd be surprised at how little overhead there is left on the elements. When used correctly, HTML's footprint on an element can be reduced significantly (and it's not very hard to use it right).

Why should I use this instead of, say, jQuery?

Personally, I usually avoid huge libraries because I never use half of the great tools provided. HTML does one job and it does it well, in a friendly way too which is always a plus.

API

HTML document.body

The global HTML is the document body and all element tags queried via the dot operator are descendants. It acts as the root HTML.

HTML.tag.. HTMLElement|Array

HTML allows you to access document nodes via their tag name. If multiple elements of the same tag list exist, an native array of elements is returned otherwise a HTMLElement (or node) is returned. Chaining is acceptable and encouraged.

HTML.div.h1; //body>div>h1. Returns h1 element.
HTML.div.h1.innerHTML = "wobblewobble"; //..

HTML.find( selector ) HTMLElement|Array

HTML.find acts as a proxy for querySelectorAll and returns a HTML'ified node or array of nodes.

HTML.find("#example"); //Single node returned
HTML.find(".example"); //Array of nodes returned if more than one, otherwise a node
HTML.find("#example").h1.em; //Chaining acceptable

HTML.tag...find( selector ) HTMLElement|Array

Scoped find's are also acceptable.

HTML.div.find(".example"); //Scoped query selector all

HTML.tag...use( callback(element) ) Root HTMLElement

Use a the current node or array of nodes within a closure. Returns the root node.

HTML.div.ul.li; //Returns a list of list items
HTML.div.ul.li.use(); //Returns div element
HTML.div.ul.li.use(function(li, i) {
	//Loop over the list items
	li.textContext = "List item #" + i;
});

HTML.div.ul.use(function(ul) {
	ul.style.background = "blue";

	//Continue using the ul
	ul.li; //..
});

HTML.tag...eq( begin , end ) HTMLElement|Array

When an array of elements is returned, eq allows you to operate on a single element in a list or a range of elements.

HTML.ul.li; //List of 10 li elements
HTML.ul.li.eq(3).innerText = "Fourth li element!";

HTML.ul.li.eq(3, 9).use(function(li, i) {
	li.create.em.innerText = "Emphasized text!";
});

//Alternatively use forEach, remember it's a native array!
HTML.ul.li.eq(3, 9).forEach(function() { ... });

HTML.create.tag.. HTMLElement|Array

The root HTML.create object. Any valid HTML tag can be queried via the dot operator on this object and an element of that tag will be created. Note: Elements created here will not be appended into the root. They are orphans. It's up to you to place them.

HTML.create.div; //Returns a div element
HTML.create.div.h1; //Returns a h1 element with div parent
HTML.create.div.h1.em; //etc

HTML.create.tag...mult( factor ) Array

During the creation process you may want more than one element. mult allows you to multiply the current node by a factor.

HTML.create.ul.li.mult(10).use(function(li, i) {
	li.innerHTML = "Created list items!";
});

//Use in conjuction with eq
HTML.create.ul.li.mult(10).eq(5).innerHTML = "The 5th item.";

HTML.create.special( tag ) HTMLElement

Due to some namespace clashes, some tag names had to be moved to a function. These tag names include: script and style.

HTML.create.special("style"); //Returns style element

HTML.tag...create.tag.. HTMLElement|Array

Elements can also be created within a scope. Whenever create is used on an element, any elements created are appended into this element as a parent. All of the above functions apply.

HTML.div.create.h1.innerHTML = "Hello world!"; //Create a h1 and appends it into the div

HTML.div.create.ul.li.mult(10).use(function(li, i) {
	li.innerHTML = "wtf is with the li elements";
});
