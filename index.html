HTML

The DOM can be your friend.

HTML is a simple, extensible way to work directly with the DOM that won't make you want to pull your hair out. HTML.min.js (1.5kb)

Input

Output

Clear and consistent

HTML lets you traverse the DOM with readable code and a consistent set of useful functions for both nodes and arrays of nodes.

Powerful, flexible Functions

HTML provides functions to let you navigate, use, and alter the DOM with ease and clarity.

Simple extendability

Much like jQuery, HTML is very easy to extend with the features you need.

FAQ

Why not just use Voyeur.js?

Actually, this started as a fork of Voyeur. But they diverged too much to ever be merged. HTML's biggest advantage is extensibility, but there are many other improvements as well. They still retain the same property access technique, and obviously, the layout and features of Adrian's excellent github.io site. Under the covers, though, the project's source is all new and improved.

Why call it "HTML"? Isn't it confusing?

No, it's self documenting. I like code that tells you what it is doing when you read it.

Which browsers does HTML support?

The modern ones: IE9+, Firefox, Safari, Chrome, Opera and the like.

How does the DOM traversal work?

We use object getter functions to expose an element's children via their tag names on itself.

What?! You expose every element's children onto itself?

Not so much. We dynamically create getters as you traverse. Any part of the document you avoid, we avoid.

Doesn't this pollute elements with lots of properties?

Unless you have really crazy document structures or go nuts with extensions, the impact is very small.

Why should I use this instead of, say, jQuery?

It's much smaller and much more direct. The DOM in modern browsers doesn't need much wrapping these days.

Where can I get code?

Github. Feel free to fork and create a pull request!

I have a suggestion, where do I put it?

I do a happy dance when you create an issue on Github.

API

HTML document.rootElement

The global HTML is the document html and all element tags queried via the dot operator are descendants. It's the root node!

HTML.tag.. HTMLElement|Array

HTML allows you to access document nodes via their tag name. If multiple elements of the same tag list exist, an native array of elements is returned otherwise a HTMLElement (or node) is returned. Chaining is fun.

HTML.head.title; //html>head>title. Returns title element.
HTML.head.title.textContent = "A Better Title!"; //..

HTML.find( selector ) HTMLElement|Array

HTML.find acts as a proxy for querySelectorAll and returns a HTML'ified node or array of nodes.

HTML.find("#example"); //Single node returned
HTML.find(".example"); //Array of nodes returned if more than one, otherwise a node
HTML.find("#example").h1.em; //Chaining, for the win!

HTML.tag...find( selector ) HTMLElement|Array

Scoped find's are also acceptable.

HTML.body.div.find(".example"); //Scoped query selector all

HTML.tag...each( callback(element) ) HTMLElement|Array

Use a the current node or array of nodes within a closure. Returns the element or list it was called on.

HTML.body.ul.li; //Returns a list of list items
HTML.body.ul.li.each(function(li, i) {
	li.textContext = "List item #" + i;
}); //Returns li elements

HTML.body.ul.each(function(ul) {
	ul.style.background = "blue";

	//Continue using the ul
	ul.li;//...
});

HTML.tag...only( begin , end ) HTMLElement|Array

When an array of elements is returned, only allows you to operate on a single element in a list or a range of elements.

HTML.body.ul.li; //List of 10 li elements
HTML.body.ul.li.only(3).innerText = "Fourth li element!";
HTML.body.ul.li.only(3, 9).each(function(li, i) {
	li.add('em').innerText = "Emphasized text!";
});

HTML.tag...add(tag-node-list HTMLElement|Array

This automatically adds the specified tag or node (or list of those) to the element or array of elements on which it was called.

HTML.body.add('div; //Returns a div element
HTML.body.add('div').add('h1'); //Returns a h1 element with div parent
HTML.head.add(document.createElement('script'));
HTML.body.add(['section','aside','nav']);// now you're getting carried away...

HTML.tag...add('emmet-abbr') HTMLElement|Array

If you are using HTML.all.js or otherwise including the HTML.emmet.js extension, you can use emmet abbreviations to add elements to the DOM. This extension supports most emmet features, except grouping and numbering. Those may be added later.

HTML.body.add('ul>li{new list item}*10');

//Use in conjuction with only
HTML.body.add('ul>li{item}*10)
	.only(function(li,i){ return i%2; })
	.each(function(li) {
		li.textContent = "odd item";
	});
