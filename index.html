HTML

The DOM can be your friend.

HTML is a simple, extensible way to work directly with the DOM that won't make you want to pull your hair out. HTML.min.js (~2kb)

Input

Output

Clear and consistent

HTML lets you traverse and search the DOM with simple, readable, consistent code for both nodes and arrays of nodes.

Powerful, flexible Functions

HTML's functions enable you to use native DOM features with brevity, clarity, and more power than ever.

Simple extendability

HTML is very easy to extend and adapt to provide just the features you need.

See our F.A.Q.!

API

HTML document.rootElement

The global HTML is the document html and all element tags queried via the dot operator are descendants. It's the root element!

HTML.tag.. HTMLElement|Array

HTML allows you to access document elements via their tag name. If multiple elements of the same tag list exist, an native array of elements is returned otherwise a HTMLElement (or node) is returned. Chaining is fun.

HTML.head.title; // returns title element.
HTML.head.title.textContent = "A Better Title!";

HTML.find( selector ) HTMLElement|Array

HTML.find acts as a proxy for querySelectorAll and returns a HTML'ified node or array of nodes.

HTML.find("#example"); // single node returned

HTML.find(".example"); // Array of nodes returned if more than one, otherwise a node

HTML.find("#example").h1.em; // chaining, for the win!

HTML.tag...find( selector ) HTMLElement|Array

Scoped find's are also acceptable.

HTML.body.div.find(".example"); // contextual search!

HTML.tag...each( callback(element,index,array) ) HTMLElement|Array

Use a the current node or array of nodes within a closure. Returns the element or list it was called on.

HTML.body.ul.li.each(function(el, i, all) {
	li.textContext = "Item" + (i+1) + " of " + all.length;
}
}); // returns li elements

HTML.body.ul.each(function(ul) {// each works on single nodes too
	ul.style.background = "blue";

	// continue using the ul in here
	ul.li;//...
}); // returns the ul

HTML.tag...each( property , arguments ) HTMLElement|Array

Get, set, or invoke the specified property of the current node or array of nodes. Returns either the array of retrieved/returned values or the element/list it was called on.

var items = HTML.body.ul.li;
var texts = items.each('textContent'); // returns array of strings

items.each('id', 'item${i}');// sets the id of each element (replaces ${i} with index)

items.each('click');// calls 'click()' on each element, returns original items

items.each('classList.add', 'foo');// adds 'foo' class to each item's classList

HTML.tag...only( filter , endSlice ) HTMLElement|Array

When an array of elements is returned, only allows you to easily filter that list by selector, index, slice, or filtering function.

var items = HTML.body.ul.li; // all li elements

items.only('.highlighted'); // li elements that match a selector

items.only(3).innerText = "Fourth li element!";// a single element

items.only(3, 9).each(function(li, i) { // a slice of li elements
	li.add('em').innerText = "Emphasized text!";
});

items.only(function(el,i) {
    return !(i % 2);
});// uses a filter function to only return the even li elements

HTML.tag...add(tag-node-list) HTMLElement|Array

This automatically adds the specified tag or node (or list of those) to the element or array of elements on which it was called.

HTML.body.add('div'); // returns a div element
HTML.body.add('div').add('h1'); // returns a h1 element with div parent
HTML.head.add(document.createElement('script'));
HTML.body.add([node1, node2, [ul1, ul2]]); // yummy recursive adding

HTML.tag...add('emmet-abbr') HTMLElement|Array

If you are using HTML.all.js or otherwise including the HTML.emmet.js extension, you can use emmet abbreviations to add elements to the DOM. This extension supports most emmet features, except grouping and numbering. Those may be added later.

HTML.body.add('ul>li{list item text}*10');

//Use in conjuction with only
HTML.body.add('ul>li{item}*10')
	.only(function(li,i){ return i%2; })
	.each('classList.add','odd');

HTML._other Node|Array

On any element, you may access the non-element nodes via ._other

var nodes = HTML.body.section._other;
var text = nodes.only(function(node){ return node.nodeType == 3; })

HTML._.fn.[name] = function;

Use this to create your own plugins that extend the DOM.

HTML._.fn.on = function(event, callback) {
	return this.each(function(el) {
		el.addEventListener(event, callback);
	});	
};
HTML.body.nav.a.on('click', goToSection);

HTML._.field[alias] = property;

You may create property aliases for each.

HTML._.field['-class'] = 'classList.remove';
HTML._.field['+class'] = 'classList.add';
HTML.find('.foo').each('-class', 'foo').each('+class', 'bar');

FAQ

So, another jQuery-lite, eh?

Not so much. This doesn't wrap or hide the native DOM for you. It exposes and enhances it, making it your ally instead of trying to protect you from it.

Why call it HTML?

Succint, self-documenting code is the ideal, and a self-descriptive name is also a great ward against feature creep. HTML is not going to do AJAX or JSON or CSS. It is about interacting with your document's HTML structure in javascript.

Which browsers does HTML support?

The modern ones: IE9+, Firefox, Safari, Chrome, Opera and the like. Throw in an es5-shim and older browsers should get everything but the dot-traversal of the DOM.

How does the DOM traversal work?

By defining lazy object getter functions for child tags. Again, we only do it lazily, on objects you access via this library, to keep the impact small.

Hey, this is like Voyeur.js...

This began as a fork but became much different and much more. The DOM traversal technique and structure of this documentation page are about all that remain of Adrian Cooney's work.
